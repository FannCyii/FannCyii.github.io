私有Pods的创建

2017.04.17 Kivan tools  热度 ℃

##私有Pods##

       Cocoapods是非常好用的一个iOS依赖管理工具，使用它可以方便的管理和更新项目中所使用到的第三方库，以及将自己的项目中的公共组件。

1.创建并设置一个私有的Spec Repo
2.创建Pod的所需要的项目并上传代码
3.创建Pod所对应的podspec文件
4.将.podspec提交到spec库中
5.私有库的使用
6.扩展
7.讨论

1.创建并设置一个私有的Spec Repo

Spec Repo是所有的Pods的一个索引，就是一个容器，所有公开的Pods都在这个里面（说白了就是一个存储着很多Pods的远程仓库）

使用了Cocoapods后它会被clone到本地的~/.cocoapods/repos目录下，可以进入到这个目录看到master文件夹就是这个官方的Spec Repo了。[注：a)自己搭建Git; b)直接使用GitHub ]

使用命令：
pod repo add MallSpecs git@192.168.10.3:MallSpecs/MallSpecs.git

成功的话进入到~/.cocoapods/repos目录下就可以看到MallSpecs这个目录了。至此第一步创建私有Spec Repo完成。

  • 2.创建Pod的所需要的项目并上传代码

git init
git remote add origin git@192.168.10.3:MallBase/MallBase.git
git add .
git commit -m "xxxxx"
git push -u origin master

上传代码后需要设置好版本tag：

$ git tag -m "first release" 0.1.0
$ git push --tags #推送tag到远端仓库

GitHub上的命令：
git tag -a 1.0.0 -m "Tag release 1.0.0"
git push --tags

  • 3.创建Pod所对应的podspec文件

生成.podspec文件使用命令：pod spec create MallBase git@192.168.10.3:MyMall/MyMallBase.git

执行完后会生成MallBase.spec文件，注意这里的.spec文件名必须和工程文件名一样。

验证spec文件正确性：pod lib lint
忽略警告：pod lib lint --allow-warnings
pod lib lint --verbose --allow-warnings --sources=ssh://git@xx.xx.xx.107:9022/xx/KISpecs.git,git@xxxx.com:xxxx/KIVSpecs.git,git@github.com:CocoaPods/Specs.git

  • 4.将.podspec提交到spec库中

向Spec库提交podspec文件前podspec必须通过验证无误。提交后 ~/.cocoapods/repos 中的spec库文件夹中就会出现提交的.podspec文件。

使用命令：pod repo push MallSpecs MallBase.podspec
忽略警告：pod repo push MallSpecs MallBase.podspec --allow-warnings

  • 5.私有库的使用

首先，在podfile文件中添加私有库源。

source 'git@192.168.10.3:MallBase/MallBase.git'

接着就和其他cocoapods库的使用一样了：

pod ‘MallBase’ ,’->0.1.0’

  • 6.扩展

    1）公有pods
      1. 注册trunk
        $ pod trunk register useremail@xxxx.com 'password' --verbose
        $ pod trunk me
      1. 创建podspec文件
      1. 将podspec文件通过trunk推送给CocoaPods服务器
        $ pod trunk push demo.podspec
      1. 开始使用

Github 公有Spec Reop ：https://github.com/CocoaPods/Specs
